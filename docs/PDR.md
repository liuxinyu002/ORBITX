# 📄 产品需求文档 (PRD) - MVP 最终版

## 1. 产品概述

* **产品名称**：桌面端 AI 工具箱 (Desktop AI Toolbox) - **首发工具：结构化数据提取器**
* **技术栈**：Tauri (Rust) + React + Shadcn UI + Tailwind CSS
* **产品定位**：高度注重隐私、**完全孤岛化 (Air-gapped)**、本地优先的桌面效率工具。用户在不打断心流的情况下，通过全局快捷键和系统无障碍 API，将碎片化信息一键结构化并累积到本地数据库中。
* **发布平台**：Windows (UIA) & macOS (AXUIElement) 双端同步发布。
* **隐私底线**：无账号系统、无云端存储、无后台自动更新、零脱敏遥测数据上报（完全孤岛模式）。

## 2. 核心架构与界面布局

采用“下钻式 (Drill-down)”架构设计：

* **工具箱大厅 (App Dashboard)**：极简的网格面板，基于 Shadcn UI 规范，展示已安装的工具（目前为【结构化提取器】）和【全局设置】。
* **提取器内部主界面**：经典左右分栏。
* **左侧**：任务列表（如：简历库、价格表）及返回大厅的导航。
* **右侧 (Data Grid)**：采用**传统底部分页 (Pagination)** 的数据表格，每页固定显示 50 条，通过 SQLite 进行后端物理分页（`LIMIT/OFFSET`），确保万级数据下的 React 渲染绝对流畅。


* **悬浮与隐形态**：平时完全隐藏，仅保留系统托盘图标。快捷键唤醒居中轻量级悬浮窗（类似 Spotlight 模式）。

## 3. 功能模块详解与核心工作流

### 3.1 AI 模型与全局配置

* **BYOK 模式**：内置 DeepSeek、智谱、OpenAI 等主流厂商快捷配置入口。
* **本地支持**：提供“自定义 (OpenAI 兼容)”选项，完美支持本地 Ollama 接口。
* **环境策略**：同时支持云端与本地模型，用户根据自身算力和资费习惯自由切换。

### 3.2 任务创建与表头管理 (Schema Definition)

* **AI 草稿生成**：用户通过自然语言描述需求，AI 自动生成初始表头（JSON Schema）。
* **可视化表单编辑器**：用户通过基于 Shadcn UI 构建的图形化界面对表单进行“增、删、改、重命名、类型调整”等精准微调。

### 3.3 无痕抓取与全局路由 (Capture & Routing)

* **底层抓取机制**：严格采用系统无障碍 API（Win: UIA / Mac: AXUIElement）读取选中文本。**坚决不操作剪贴板**，杜绝剪贴板污染。
* **降级策略（严格模式）**：若在特殊软件中 API 读取失败，直接放弃并弹 Toast 提示“无法从当前应用获取”，不进行剪贴板模拟兜底。
* **双全局快捷键体系**：
* **快捷键 A（静默提取）**：选中文本后按下，直接抓取数据并派发给“当前已激活”的任务。
* **快捷键 B（全局命令面板）**：唤出居中的搜索下拉框。核心功能为“提取选中文本 + 分发指定任务”，并在后台将该任务设为新的激活态。



### 3.4 异常处理与长文本保护 (Error Handling & Guardrails)

* **大文本硬阻断策略（单层刚性防护）**：
* **控制逻辑**：为了引导用户保持“专注精准提取”的习惯并保护 Token/本地显存，设定刚性字数阈值（建议 **2,000 ~ 3,000 字**）。
* **拦截时机**：采用“派发时拦截”**逻辑。按下快捷键 B 时正常唤起面板并支持文本预览，仅在**“确认选择任务并点击派发/执行静默派发”的瞬间由前端计算字符长度。一旦超长，直接阻断向 LLM 发送请求，并弹出强提示：*“⚠️ 选中的内容过长，请缩减范围后重试。”*


* **智能相关性降级（依托 `pi agent`）**：在底层的 Schema 中强制 AI 返回 `is_relevant` 参数。
* **静默入库**：若 AI 判定内容与任务相符，直接静默存入本地 SQLite，仅弹 Toast 提示成功。
* **自动打断**：若用户误选无关内容（或 AI 提取失败），主动打破静默模式，弹出“预览卡片”交由用户人工确认。



### 3.5 数据管理与消费 (Data Management)

* **静态导出**：不进行深度追踪溯源（纯文本入库），用户可在 Data Grid 界面一键导出当前表格为 `.csv` 或 `.xlsx` 文件，融入其他办公流。
